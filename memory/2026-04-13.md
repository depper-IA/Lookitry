## Logica de Descripcion IA Automatica (14:30)

### Proceso Oculto
1. Usuario sube imagen de producto
2. Sistema auto-detecta categoría usando IA (via `/api/products/describe-ai`)
3. Genera descripción precisa para el probador virtual
4. Todo sucede AUTOMATICAMENTE - usuario NO ve botones ni indicadores

### Archivos Modificados
- `frontend/src/components/dashboard/ProductForm.tsx`
  - UI de "Describir con IA" eliminada (proceso automático silencioso)
  - Campo de descripción sigue disponible pero sin marca de IA
  - Categoría se detecta automáticamente sin mostrar badge

### Categorias Internas vs Publicas
**Publicas** (todos los usuarios):
- tshirt, vestido, zapatos, camisa, hoodie, chaqueta, pantalones, accesorios

**Internas** (solo marcas específicas):
- Rines: solo Wilkie Devs (`cf95f272-8de9-45e2-9290-d026312f2c31`)

### Base de Datos
```sql
ALTER TABLE category_attributes ADD COLUMN is_internal BOOLEAN DEFAULT false;
ALTER TABLE category_attributes ADD COLUMN allowed_brands UUID[];
```

### Flujo de Producto
1. Admin sube imagen
2. AI genera descripción + categoría automáticamente
3. Admin ve solo campos normales (sin indicadores IA)
4. Campos guardados: description, category, short_description, attributes
