# Estado del Proyecto: v0.9.0-beta.1 (Beta Comercial)

## Fase Actual: Beta Comercial
**Fecha de Referencia:** Marzo 2026
**Objetivo:** Validar conversión real y estabilidad con clientes piloto.

---

## Hoja de Ruta (Roadmap)

### Corto Plazo (0-6 semanas) — Monetización y Estabilidad
- [ ] Cerrar flujo de checkout 100% confiable (Wompi + PayPal).
- [ ] Reducir fricción en onboarding (registro post-pago simplificado).
- [ ] Implementar telemetría básica de KPIs: conversión, activación y churn.
- [ ] Congelar features no críticas para asegurar estabilidad.
- [ ] Definir arquitectura base del plugin de WooCommerce.

### Mediano plazo (6-16 semanas) — Plugin MVP y Crecimiento
- [ ] Lanzar Plugin MVP: Autenticación con API Key y Sync básico de catálogo.
- [ ] Mejorar observabilidad de errores por tienda (Merchant Monitoring).
- [ ] Campañas de adquisición ligera basadas en feedback real.
- [ ] Evolución a v0.10.x con los primeros clientes en producción.

### Largo plazo (4-9 meses) — Escalar
- [ ] Plugin robusto con Sync incremental y Auto-updates.
- [ ] Automatización de lifecycle marketing (emails, dunning, upgrades).
- [ ] Optimización de costos de infraestructura e IA.
- [ ] Preparación para v1.0 (Estabilidad y Retención confirmadas).

---

## Regla de Enfoque
Toda tarea nueva debe responder: **"¿Esto mejora ingresos o retención en los próximos 30-60 días?"**. 
Si la respuesta es negativa, se delega al backlog de mediano/largo plazo.
